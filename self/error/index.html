# 心法数则

不要自责，人难免不摔倒。

# 法则一:不要我以为

在工作中涉及到团队协作的事情，不要"我以为"，所有的错误都建立在自己的自以为是的愚蠢为前提。“我以为、我认为、我没想到...”这些统统是你捅娄子后为了让面子上好看点的遮羞布。

# 法则二:以诚做工，以工做诚

对待你领域的工作，你应该饱满状态去解决当下问题，而不是玩玩打打吊儿郎当满不在乎。当你诚心诚意的做漂亮自己的工作，相信我它不会辜负你。或者你没那么幸运，你的领导是个昏君将你雪藏深山老林，请你不要灰心，老天会有所眷顾。

# 法则三:不要出风头

木秀于林，风必摧之。不管你在哪，请你不要太狂妄自大。也不要招摇过市，相信我低调是永远不过时的智慧

# 法则四:将充裕的时间，紧迫化

意外和惊喜，你不知道谁先来。你觉得这次任务虽然重，但时间很充裕，你可以悠哉游哉的来做。相信我这是不明智的，因为等你来了一个很急的任务，此刻你将被动无奈的加班。7 天迭代是最佳时间，如果是功能非常庞大的，我建议将 7 天提升至 14 天，不能再多了。

# 法则五:愚公移山，做好样板

人生难免不需要移动几座大山来重构世界，在这之前一定要规划好业务边界、接口历史、接口调用方、接口参数，最后的迁移方案撰写，不要盲目动工。井井有条才是真!

# 法则六:对于未知，请你警惕

当你使用第三方组件或者是同事开发的组件，请注意使用方式避免不必要的趟浑水。
